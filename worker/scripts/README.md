# Worker Provisioning Scripts

Run `provision.sh` once from any directory to create the two KV namespaces (`TOKEN_CACHE`, `DEAD_LETTER`), substitute their IDs into `wrangler.toml`, and print exact instructions for the remaining manual steps. Full environment variable and binding specifications live in `../SECRETS.md`; GitHub App creation and private-key conversion procedures are in `docs/ops/github-app-setup.md` (project root). Before deploying, the operator must also replace `<RATE_LIMIT_NAMESPACE_ID>` in `wrangler.toml` with the namespace ID from the Cloudflare dashboard, then register three secrets interactively (`wrangler secret put GH_APP_PRIVATE_KEY`, `wrangler secret put RESEND_API_KEY`, `wrangler secret put TURNSTILE_SECRET_KEY`), and set two vars directly in `wrangler.toml` (`GH_APP_ID` and `PRODUCT_ROUTES`). Once all placeholders are replaced and secrets are registered, a single `wrangler deploy` activates the Worker, the Cron trigger, both KV bindings, and both Rate Limiting bindings.
